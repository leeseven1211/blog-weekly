# Gemini 评估｜学习阮一峰最近 100 期后的周刊改进建议

> 日期：2026-04-27  
> 样本：阮一峰《科技爱好者周刊》issue 295-394；对照《小七的周刊》第 012 期。

## 1. 阮一峰最近 100 期可学习模式

- **周刊是图文流，不是深度报告。** 最近 100 期每期图片均值约 44 张，中位数 44 张；读者靠滑动中的图片判断是否停下来读。
- **封面是三维世界锚点。** 封面几乎都是真实世界照片：建筑、风景、奇特装置、日常奇观，而不是产品 UI、概念海报或大字报。
- **封面图注很轻。** 主要交代“地点 + 是什么 + 一句朴素观察”，不强行把整期观点压进封面。
- **栏目顺序极稳定。** 常见顺序是：封面图 → 1-5 个前置短主题 → 科技动态 → 文章 → 工具 → AI 相关 → 资源 → 图片 → 文摘 → 言论 → 往年回顾。
- **头轻、身广。** 卷首不是千字长文，而是几个短杂谈 / 短主题，随后进入高密度条目。
- **小标题口语化。** 标题像朋友转述一个观点：“脑腐状态”“测试是新的护城河”，而不是“某公司发布某产品”。

## 2. 第 012 期当前差距（Gemini 评分）

- 内容：8/10。洞察不错，但主题太集中在 AI 落地，缺少杂感和广度。
- 标题：6/10。偏深度专栏，不够轻松、有钩子。
- 图片：4/10。图片密度低，且多为产品/架构图，缺少真实生活视角。
- 节奏：3/10。致命问题是卷首“封面主题”近千字，头太重，不像轻量周刊。
- 读者记忆点：7/10。观点有记忆点，但视觉和栏目辨识度不足。
- 整体完成度：6/10。作为深度科技观察文章不错；作为周刊，还不能和阮一峰有一拼。

## 3. 下期设计方案

### 推荐结构

1. 封面图：真实世界照片 + 轻图注，图注不超过 100 字。
2. 本周短谈：2-3 个独立短主题，每个 200-300 字，必要时各自配图。
3. 科技动态：4-6 条，每条一张图，标题观点化。
4. 工具与开源：4-6 条，每条真实 UI / repo / CLI 截图。
5. 文章与资源：可纯文本，推荐语短。
6. Moltbook 社区观察：作为软性结尾，少量精选，不堆列表。
7. 言论 / 往年今日 / 冷知识：碎片化收尾，降低阅读压力。

### 图片策略

- 封面只用真实照片：建筑、风景、人文奇观、工程装置、日常奇观。
- 动态优先现场图、发布会图、产品实图、硬件实拍、数据图。
- 工具优先真实 UI、终端、README 截图、配置片段，拒绝 logo / banner。
- 下期目标图片数至少 20-30 张；每滑动 1-2 屏应出现一张有信息量的图片。

### 封面 checklist

- [ ] 是否是真实世界存在的物理对象？
- [ ] 是否一眼有奇特感、美感或故事性？
- [ ] 图注是否能用“这是某地 + 某物”说清？
- [ ] 是否没有车辆、路标、施工感、断桥感等干扰隐喻的元素？
- [ ] 是否不用产品 UI、概念图、AI 渲染、大字海报？

### 发布前验收 checklist

- [ ] 最大文字块是否超过手机一屏 / 约 300 字？超过就拆。
- [ ] 是否每 1-2 次滑动就能看到一张真实图片？
- [ ] 小标题是否像周刊短评，而不是新闻通稿？
- [ ] 工具条目是否直接讲“它能干什么、长什么样”，没有字段表味？
- [ ] 整体是不是“开眼界、无压力、易滑动”？

## 4. 应写入长期记忆的硬规则

1. 卷首导读禁止千字长文；单块超过 300 字就拆。
2. 周刊要“头轻身广”：短主题 + 高密度条目，不做单主题报告。
3. 封面只用真实世界照片；产品 UI / 抽象概念 / AI 渲染 / 大字海报不得做封面。
4. 封面图注要白：地点、对象、来源、轻观察，不强行升华。
5. 科技动态和工具尽量一条一图；没有好图宁可不上。
6. 小标题必须短、口语化、有观点，废弃“某公司发布某产品”的通稿标题。
7. 工具栏第一眼要看到真实界面 / 终端 / repo，不要 logo 或官网 banner。
8. 写作骨架不能暴露成正文模板字段。
9. 每期保留 1-2 条非 AI / 非 IT 的“无用之用”，给读者呼吸感。
10. 增加条目数量，降低单条字数；读者是来逛周刊，不是来上课。
